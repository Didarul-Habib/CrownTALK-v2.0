CT_ANGLE_ONE_LINER: Beyond positions itself as a native Bitcoin interoperability protocol that bridges assets to, from and within Bitcoin L1, including cross-standard conversion between multiple Bitcoin token standards.

RISKS_SHORT:
- Bridge-risk class is historically high-risk; Beyond uses multi-component lock/mint patterns that have been exploited across crypto.
- Permissioned MPC/multisig “Forge” (11 signers, 7-of-11) concentrates trust and adds governance/censorship/availability risk until V2.
- Token/incentive timelines appear in UI but are still evolving (“not listed” elsewhere); leaderboard incentives can attract farming.

COMPARABLES:

===============================================================================
1) CORE VALUE PROP
===============================================================================
- Problem: Bitcoin’s on-chain “economy” is small and fragmented versus its market cap, with multiple token standards and L2s creating friction for builders and users.
- Solution: Beyond proposes a tri-directional bridge that (1) brings external assets (e.g. stablecoins, majors) into Bitcoin as L1-native representations, (2) exports Bitcoin-native assets to other chains, and (3) supports within-Bitcoin moves across token standards.
- Differentiation: Emphasis on cross-standard conversion on Bitcoin L1 itself, plus LayerZero-powered omnichain connectivity to many non-Bitcoin networks.
- Why now: The rise of inscriptions/ordinals and competing Bitcoin token standards makes fragmentation more painful; BTCFi narratives create demand for secure, “standard-aware” bridges.

===============================================================================
4) MAIN RISKS
===============================================================================
- Bridge-design / smart contract risk:
  - Implements a “lock on source / mint on destination” bridging pattern with multiple on-chain and off-chain components—a class historically associated with major exploits.
- Custody, multisig and centralisation risk:
  - Bitcoin-side “Forge” component is described as an MPC-backed multisig with 11 signers and a 7-of-11 threshold; V1 is explicit about being permissioned, with a V2 plan toward staking-based permissionless operation.
  - Guardian nodes and “private-key PoA multisig” approaches concentrate trust and introduce governance/censorship/availability questions.
- Security process maturity:
  - Docs reference at least one third-party audit (e.g. Zokyo, completed Q4 2024); scope/recency should be rechecked near mainnet.
  - Bug bounty intent via Immunefi is tied to mainnet going live; verify once active.
- Token / incentives risk (if $BYD proceeds):
  - $BYD appears in the UI while external data sources flag the token as “not listed,” suggesting status/timelines are evolving.
  - Leaderboard incentives can attract mercenary usage and wash-like patterns without strong anti-sybil/anti-abuse rules.

6) Security Model & Trust Assumptions
──────────────────────────────────────────────────────────────────────────────
Security principles listed in docs
- Encryption (TLS for transit, AES for data at rest).
- Key management with MPC + threshold signing (TSS), splitting control across signers.
- Smart contract security emphasis (audits, pre-deploy review).
- Node security (hardening, monitoring), secure APIs, access controls, intrusion detection, DDoS protection.
- Logging/monitoring and incident response plan.
- Continuous security assessment and penetration testing.

Critical trust points (interpretation of the model)
- Forge custody & signer set is a major trust surface:
  - An MPC/multisig model reduces single points of failure but introduces governance/operational risks (signer integrity, key management procedures, incident response).
- Reliance on cross-chain messaging layer:
  - Beyond publicly ties part of its trust model to LayerZero-based messaging standards and security posture.

Audits
- Beyond’s docs list audits; one public reference is a Zokyo audit report dated Oct 30, 2024 (linked from Beyond docs and listed in Zokyo’s reports index).

Bug bounty
- Docs point to an Immunefi-based bug bounty program, but also state “once Beyond goes live on mainnet” it will launch with a significant, permanent program (i.e., it may not be fully live yet).
