PROJECT_POST_CARD_V1
====================

PROJECT_ID: WALLCHAIN
SLUG: wallchain
NAME: Wallchain (Wallchain Quacks / X Score)
TICKER: N/A (Quacks are documented as points, not a confirmed public token ticker)
PRIMARY_CHAIN: Multi-surface (X-native scoring + optional “onchain footprint” signals; not TVL-first)
CATEGORY: AttentionFi / InfoFi (mindshare + reputation + growth analytics)
STAGE: Live (daily-updated leaderboards + live Chrome extension)

ONE_LINE_PITCH:
An AttentionFi platform that scores real crypto influence on X (X Score) and rewards quality contributions via Quacks + mindshare leaderboards.

CORE_NARRATIVE:
- “Quality-weighted influence” vs vanity metrics: graph-based signals to reduce fake followers / engagement gaming.
- “Mindshare as a measurable asset”: track who actually drives meaningful discussion around projects/topics.
- “Creator-first growth layer”: reward consistency + useful posting, not spammy farming.
- Two-phase story: legacy MEV/anti-bot infrastructure → now repositioned around growth/reputation tooling.

PRODUCT_DETAILS:
- X Score (0–1000): computed from a Crypto Twitter Connectivity Graph (CTCG) using actions like follows/retweets/comments; includes anti-manipulation logic (e.g., decay).
- Quacks: daily-calculated reward points for valuable/insightful/engaging content; leaderboards update daily (roughly ~11:00 UTC).
- Quacks App UX: login with X → profile shows X Score + Quacks balance → unlock “personal mindshare leaderboard” → filters (7d/30d/3 months/all time) → browse partner leaderboards.
- Chrome extension (“Wallchain X Score”): shows scores in-product; docs indicate you can obtain a score without installing, but extension helps view/track.
- Referral loop: users earn 10% of Quacks earned by referred accounts.
- Anti-farming stance: bot/farming detection + enforcement (including potential permanent bans).
- B2B growth tooling: position as influence discovery + campaign/leaderboard launches + APIs (mindshare tracking/heatmaps/X Score).
- Publicly shown campaign examples/case studies include: Limitless, HeyElsa, Avantis.

KEY_METRICS:
- TVL: Not a primary KPI for AttentionFi; publicly listed vault addresses (where referenced) show negligible balances (directional only, can change).
- Distribution proxy: Chrome extension lists ~20,000 users and a strong store rating; shows a recent update date (Jan 15, 2026).
- Social footprint: @wallchain has a large follower base; X Score-specific account also exists.
- Operational cadence: Quacks calculated ~once/day; leaderboards updated daily around ~11:00 UTC.
- Coverage: Wallchain states support for 23 languages (explicitly including Bengali).

RISKS_AND_REALISM:
- Platform dependency risk: scoring + rewards depend on X data access and Wallchain’s own infra/models; policy/rate-limit shifts can impact reliability.
- Centralization risk: Wallchain controls scoring methodology, leaderboard logic, enforcement, bans, and partner campaign definitions.
- Incentive design risk: any rewards/leaderboards system attracts farming attempts; enforcement can create false positives/negatives and ranking volatility.
- Token speculation risk: docs describe Quacks as points; avoid assuming token/TGE unless officially published.
- Smart contract risk (contextual): Wallchain has shipped onchain components historically; third-party audits exist but don’t eliminate risk; some designs include privileged/admin controls.

CT_ANGLES_HOW_TO_TALK:
- Position it as a “signal layer” for CT: helps separate real influence from vanity metrics.
- Lean into “quality + consistency” framing: Quacks reward useful posting and sustained contribution.
- For growth teams: “better KOL discovery + campaign measurement” without pretending it’s perfect.
- Mention credibility carefully: long arc from MEV/anti-bot infra + accelerator/backing; don’t turn it into guaranteed alpha.
